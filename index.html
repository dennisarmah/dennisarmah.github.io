HRIS Access Control Audit

IT General Controls (ITGC) audit · Apex Horizon (simulated organization) · 02 Mar – 30 Apr 2026
Best in Cohort

Led a full-cycle ITGC audit of a Human Resource Information System, testing user provisioning/de-provisioning, privileged access management, and authentication controls against ISO/IEC 27001:2022, NIST SP 800-53, and COBIT 2019. Sampled 35 active accounts, 15 terminated-employee records, and 100% of the 10 privileged accounts, then ran Test of Design and Test of Effectiveness procedures on each control area.

Control IDAreaFindingRisk
AH-HRIS-01 User Provisioning 5 terminated-employee accounts stayed active past the deactivation window (avg. 8-day delay) High
AH-HRIS-02 Privileged Access 3 admin accounts had elevated access with no documented approval or periodic review High
AH-HRIS-03 Authentication Weak password policy: 8-character minimum, no account lockout configured Medium

Delivered a Corrective Action Plan with named owners and target dates: automated deprovisioning tied to HR exit workflows, a formal privileged access approval process with quarterly reviews, and a 12-character password policy with lockout after 5 failed attempts, plus a full management summary report for governance review.

Enterprise IT Risk Assessment

Microsoft Exchange Email Server & Oracle Database System · GRC risk-assessment engagement
Risk Register

Assessed two critical systems (messaging infrastructure and a production database) using a likelihood × impact model aligned to ISO 27001, ISO 31000, and the NIST Cybersecurity Framework. Built full risk registers per system, rated inherent vs. residual risk, and proposed concrete mitigations mapped to each finding.

Risk IDSystemRiskRating
EX01 Exchange Phishing: credential theft via external email Critical
EX02 Exchange Unauthorized access from weak credentials Critical
DB01 Oracle DB Unauthorized access via weak access controls Critical
DB02 Oracle DB SQL injection from unvalidated application input High

Recommendations included a Zero Trust access model, MFA and Conditional Access via Microsoft Entra ID, Oracle Database Vault for privileged data, parameterized queries with SAST/DAST testing, and centralized monitoring through Microsoft Sentinel, with RTO/RPO targets formalized for both systems.

Cryptography & Secure Communications Lab

Hands-on practicum: hashing, symmetric/asymmetric encryption, SSL/TLS validation, WhatsApp E2E encryption
Hands-On Lab

Completed a four-part hands-on lab covering the cryptographic controls that protect data at rest, in transit, and in messaging. Each part paired a real tool or live website with a written explanation of the underlying security mechanism, the same way I'd document a control test for an audit workpaper.

TopicTool / MethodKey Takeaway
Hashing WinMD5 checksum comparison Any change to a file's content changes its MD5 hash entirely, confirming hash-based tamper detection for file integrity
Symmetric Encryption Hat.sh client-side encryption One shared key both encrypts and decrypts; the key itself becomes the weak point if shared over an insecure channel
Asymmetric Encryption Hat.sh public/private key pairs Encrypting with the recipient's public key and signing with the sender's private key removes the shared-key exposure risk entirely
SSL/TLS Browser certificate inspection Verified certificate issuer, validity window, and domain match before trusting a site for financial transactions
E2E Messaging WhatsApp protocol analysis Mapped the handshake: a per-message session key gets wrapped in the recipient's public key so only their private key can unlock it

The lab reinforced a point that carries directly into audit work: access controls alone aren't enough. A system can have flawless provisioning and still leak data in transit without validated TLS and properly implemented encryption, which is why cryptographic controls get tested as their own control domain.
